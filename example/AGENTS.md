# Agent Instructions

This project's AI guardrails are maintained by [VibeTags](https://github.com/PIsberg/vibetags)
in **[CLAUDE.md](./CLAUDE.md)**. Refer to that file for the full, generated guardrail set
(locked files, contextual rules, security audits, PII guardrails, and more).

> **Why isn't this file generated?**
> VibeTags only auto-manages `AGENTS.md` when it is the *sole* AI config file in the project.
> Because this example also ships `CLAUDE.md`, `.cursorrules`, and many other AI config files,
> `AGENTS.md` is treated as a hand-authored pointer and is intentionally left untouched.
> Keep only `AGENTS.md` (and remove the other AI config files) to have VibeTags generate it.
