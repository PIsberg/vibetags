Three optimisations land in 0.7.1, all targeting steady-state incremental builds:

  1. WriteCache (.vibetags-cache sidecar): skips read+compare when the cache says
     we wrote the same body to a byte-stable file (size + mtime). isUnchanged
     uses one Files.readAttributes call (size + mtime) instead of two getXxx
     syscalls. recordWrite reuses body.getBytes(UTF_8).length instead of a
     post-write Files.size syscall. Net write-side overhead: ~1 stat + sha256.

  2. Streaming byte-compare for non-marker writes where existing file size
     matches new content size exactly. Replaces Files.readString +
     .strip().equals() with an early-exit byte-by-byte stream.

  3. Pre-sized per-platform StringBuilders in GuardrailContentBuilder
     (cursorRules / claudeMd / codexAgents / copilot / qwenMd / windsurfRules /
     zedRules / llmsTxt / llmsFullTxt). Sized at ~1500 chars per annotated
     element, capped at 256 KB.

Caveats — read before drawing conclusions:

- The JMH suite calls writeFileIfChanged directly, bypassing
  GuardrailContentBuilder. Phase 3's win is invisible there.
- The smallWrite benchmark mutates the file before each iteration
  (Files.writeString(smallFile, "x") then writeFileIfChanged with 1024 bytes),
  so the cache always misses → Phase 1's hit-path benefit is invisible.
  What 0.7.1 buys you in this benchmark is parity with 0.7.0 plus the cache
  bookkeeping cost being trimmed close to zero by reusing the body length.
- The stress sweep creates a fresh TempDir per leg → cache is empty → no hit
  path to measure. Numbers reflect Windows process-launch jitter ±15%.

How to actually see the wins:

- example/ project: clean build creates .vibetags-cache; second mvn compile
  leaves all generated file mtimes untouched (cache hit on every file, zero
  reads, zero writes).
- A future JMH benchmark that warms the cache before measurement (e.g. with
  -wi 5 on a noChange variant whose @Setup pre-populates the cache) would
  show the hit path's ~150 us savings vs. the historical readString path.
